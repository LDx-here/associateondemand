# AssociateOnDemand Project DNA: The Vision and Context

**Author:** Manus AI
**Date:** May 2, 2026
**Project:** AssociateOnDemand powered by RMV

## 1. Project Overview: AssociateOnDemand (powered by RMV)

AssociateOnDemand is envisioned as an autonomous law firm platform and a "Decision Engine" designed for high-volume legal practices. Its primary goal is to automate critical legal workflows, including intake, case assessment, and document generation, leveraging advanced AI agents. Initially, the platform targets the immigration and personal injury legal domains.

## 2. Strategic Pivot: Airtable-First Architecture

The project has undergone a significant strategic pivot from a complex Next.js/FastAPI/PostgreSQL stack to a simplified **Airtable-backed architecture**. This change was implemented to accelerate development, reduce backend complexity, and enhance data management flexibility. Airtable now serves as the primary backend-as-a-service, providing relational data capabilities and an intuitive interface.

## 3. Current State and User Experience Goals

Currently, the project is in a transition phase where the user is seeking to simplify execution and move towards a more user-friendly interface. The UI has evolved from a "Neo/Matrix" theme to a **Professional Enterprise aesthetic** (Legora/Clio/E-immigration style). The ultimate goal is to achieve a **"Harvey-style" clean, chat-centric interface** that maintains professional dashboard functionalities.

## 4. Technical Context

The core technologies underpinning AssociateOnDemand include:

*   **Frontend:** Next.js (React, TypeScript)
*   **Styling:** Tailwind CSS
*   **Data Management:** Airtable (Backend-as-a-Service), TanStack Table (for data display)
*   **Iconography:** Lucide Icons
*   **Automation & Integration:** Zapier/Make (for autonomous workflows and AI agent integration)
*   **Document Processing:** OCR (e.g., AWS Textract for the "Strong Reader")
*   **Knowledge Management:** Obsidian (for bi-directional sync)

## 5. Key User Requirements and Future Directions

The development of AssociateOnDemand is guided by several key user requirements:

*   **Professional Aesthetic:** A clean, enterprise-grade user interface, moving away from previous thematic designs.
*   **Airtable-Based Data Management:** Leveraging Airtable for its flexibility and ease of use in managing legal data.
*   **"Harvey-Style" Interface:** A clean, chat-centric interaction model for intuitive user engagement.
*   **Dual-Pathway Intake:** Support for both Immigration and Personal Injury case intake processes.
*   **Automated Case Auditing:** Implementation of logic for automated case assessment.
*   **Fact-to-Legal Element Mapping:** A crucial AI-driven feature to translate extracted facts from documents into relevant legal elements.
*   **Bi-directional Obsidian Sync:** Integration with Obsidian for knowledge management and legal research.
*   **Autonomous Workflows:** Utilizing Zapier/Make to automate processes and integrate AI agents.

The goal is to build a robust, scalable, and user-friendly platform that revolutionizes legal practice management.
